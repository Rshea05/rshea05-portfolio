# GoDaddy Portfolio Redesign Notes

## Recommended Structure

Use the Work / Portfolio page as a vertical case-study page instead of a sideways gallery.

1. Intro section
   - Eyebrow: Selected UX Work
   - Main headline: Design systems, product strategy, and workflows that help teams ship with clarity.
   - Short paragraph explaining your positioning at the intersection of UX strategy, product delivery, design systems, and AI-enabled workflows.

2. Focus areas
   - UX Strategy
   - Design Systems
   - Design Operations
   - AI Workflows

3. Vertical case studies
   - Use one full-width row per project.
   - Put a project image, prototype screenshot, diagram, or blurred-safe work sample on the left/top.
   - Put the case-study content on the right/bottom.
   - For each case study, include:
     - Project type tags
     - Title
     - Two- to three-sentence summary
     - Role
     - Focus
     - Impact
     - Button to contact you or view supporting material

4. Process section
   - Frame the problem
   - Structure the work
   - Align and iterate
   - Ship repeatable quality

## GoDaddy Builder Steps

1. Open the Work - Portfolio page in GoDaddy Website Builder.
2. Remove or replace any gallery/carousel section that forces horizontal scrolling.
3. Add a standard vertical section for the page intro.
4. Add a four-column content block for the focus areas. On mobile, GoDaddy should stack these automatically.
5. Add one repeated content/image section per case study. Choose a layout with image on one side and text on the other for desktop, stacked layout for mobile.
6. Add headings for Role, Focus, and Impact under each project. This is a common UX portfolio pattern and helps recruiters scan quickly.
7. Add real visuals where possible: product screens, journey maps, system diagrams, before/after flows, FigJam/Figma snapshots, or safe anonymized artifacts.
8. Preview mobile and desktop before publishing. Confirm the page scrolls vertically and that no section requires dragging sideways.
